# Руководство по проекту «Шебер»

Этот файл содержит расширенную русскоязычную инструкцию по запуску, проверке и дальнейшей разработке проекта.

## Требования

Достаточно современного браузера с поддержкой `dialog`, `localStorage`, `sessionStorage`, `FileReader` и ES6. Установка пакетов не требуется.

## Локальный запуск

1. Склонируйте репозиторий и перейдите в его каталог.
2. Запустите любой статический HTTP-сервер, например:

   ```bash
   python -m http.server 8000
   ```

3. Откройте `http://localhost:8000` в браузере.

Прямое открытие `index.html` также возможно, но HTTP-сервер обеспечивает более предсказуемую работу браузера.

## Основные сценарии проверки

1. Найдите услугу через строку поиска.
2. Переключите город, категорию и способ сортировки.
3. Создайте демонстрационный аккаунт.
4. Добавьте объявление и при желании прикрепите JPG, PNG или WEBP до 2 МБ.
5. Добавьте карточку в избранное и откройте страницу объявления.
6. Выйдите из аккаунта и проверьте защиту пользовательских действий.

## Хранение данных

Приложение не отправляет введённые данные на сервер. В браузере используются ключи с префиксом `sheber`:

- `sheberUsers` — демонстрационные аккаунты;
- `sheberSession` — текущая сессия;
- `sheberListings_<ID>` — объявления пользователя;
- `sheberFavorites_<ID>` — избранное пользователя.

Чтобы полностью сбросить демонстрационные данные, очистите локальное хранилище сайта через инструменты разработчика браузера.

## Ограничения демонстрационной версии

- пароли хранятся локально без серверной защиты;
- объявления доступны только в том браузере, где они созданы;
- карточка объявления передаётся новой вкладке через `sessionStorage`;
- удалённые изображения и шрифт требуют подключения к интернету;
- серверная модерация, база данных и восстановление пароля отсутствуют.

## Разработка

Проект не имеет шага сборки. После изменения HTML, CSS или JavaScript обновите страницу браузера. Перед выпуском версии вручную проверьте оба размера экрана, основные сценарии из раздела выше и актуализируйте `CHANGELOG.md`.

Версии оформляются семантическими тегами `vМАЖОРНАЯ.МИНОРНАЯ.ПАТЧ`, например `v1.0.0`.
