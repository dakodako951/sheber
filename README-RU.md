# Руководство по проекту «Шебер»

## Требования

- Node.js 22.5 или новее — используется встроенный модуль `node:sqlite`;
- npm;
- современный Chrome, Firefox или Edge.

## Установка и запуск

```bash
git clone https://github.com/dakodako951/sheber.git
cd sheber/backend
npm ci
npm start
```

После запуска откройте `http://localhost:3000`. Сервер одновременно предоставляет API и раздаёт файлы из `frontend/`, поэтому отдельный статический сервер не требуется.

Для разработки с автоматическим перезапуском:

```bash
npm run dev
```

Проверка синтаксиса сервера:

```bash
npm run check
```

## API

- `GET /api/health` — состояние приложения и базы;
- `/api/auth/*` — регистрация, вход, сессия и выход;
- `/api/listings` — каталог и управление объявлениями;
- `/api/profile` — профиль пользователя;
- `/api/favorites` — избранные объявления.

## Хранение данных

SQLite-файл `backend/stroy.db` создаётся при первом запуске. Для сохранения данных на хостинге каталогу `backend/` требуется постоянный диск. Файл базы, WAL-файлы и локальный `.env` не должны попадать в Git.

Пароли в базе не хранятся открытым текстом: применяется `scrypt` и случайная соль. Сессионные токены генерируются через `crypto.randomBytes`.

## GitHub Pages

GitHub Pages автоматически публикует только `frontend/`. Это статическая демонстрация интерфейса. Pages не поддерживает Node.js и SQLite, поэтому для регистрации и общей базы backend нужно разместить отдельно или запускать локально.

## Основные проверки

1. Откройте `/api/health` и убедитесь, что возвращается `status: ok`.
2. Зарегистрируйте пользователя и повторно войдите.
3. Создайте, отредактируйте и удалите своё объявление.
4. Проверьте запрет изменения чужого объявления.
5. Проверьте личный кабинет, избранное и выход.
6. Проверьте страницы при ширине мобильного экрана.

Версии оформляются семантическими тегами вида `v3.0.0`.
