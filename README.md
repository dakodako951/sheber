# Шебер — сервис поиска мастеров

«Шебер» — демонстрационный веб-сервис объявлений для поиска специалистов по ремонту и бытовым услугам в Казахстане. Проект работает полностью в браузере и не требует сборки или серверной части.

## Возможности

- поиск мастеров по тексту, городу и категории;
- сортировка по популярности, рейтингу и цене;
- регистрация и вход в демонстрационный аккаунт;
- добавление, просмотр и удаление собственных объявлений;
- редактирование собственных объявлений из каталога и со страницы карточки;
- облачные аккаунты и общая база объявлений на Supabase;
- личный кабинет с объявлениями, избранным и настройками профиля;
- загрузка фотографии работы размером до 2 МБ;
- избранные объявления и отдельная страница карточки мастера;
- адаптивная вёрстка для компьютеров и мобильных устройств.

Данные аккаунтов, объявлений и избранного сохраняются только в `localStorage` текущего браузера. Это демонстрационный проект: настоящего сервера, платежей и защищённой авторизации в нём нет.

## Быстрый запуск

Для обычного просмотра откройте `index.html` в современном браузере. Рекомендуемый вариант — запустить локальный HTTP-сервер из каталога проекта:

```bash
python -m http.server 8000
```

После этого откройте `http://localhost:8000`.

## Структура проекта

- `index.html` — каталог, формы входа и добавления объявления;
- `data.js` — общие демонстрационные объявления для обеих страниц;
- `app.js` — поиск, фильтры, аккаунты, объявления и избранное;
- `style.css` — основные стили;
- `listing.html`, `listing.js`, `listing.css` — страница отдельного объявления;
- `логитип/` — графические материалы проекта;
- `README-RU.md` — подробная инструкция для пользователей и разработчиков;
- `CHANGELOG.md` — история версий.

## GitHub Pages

Проект публикуется автоматически из ветки `main` с помощью workflow `.github/workflows/pages.yml`. Сборка не требуется: GitHub Pages получает готовые статические файлы из корня репозитория. Файл `.nojekyll` отключает ненужную обработку Jekyll.

## Технологии

HTML5, CSS3 и чистый JavaScript без сборщика. Для облачной авторизации и общей базы используется Supabase, для шрифта — Google Fonts, для демонстрационных фотографий — Unsplash.

## Настройка Supabase

Структура таблиц, Row Level Security и бакет изображений описаны в `supabase-schema.sql`. SQL выполняется один раз в редакторе Supabase. В `backend.js` используется только публичный publishable key; доступ к данным ограничивается политиками RLS. Service Role Key нельзя добавлять в репозиторий.
